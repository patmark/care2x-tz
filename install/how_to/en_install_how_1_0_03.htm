Installation

Semi-automatic

NOTE: The following installation procedure is valid ONLY for the beta 1.0.03.

If your system has a built-in Apache-PHP-mySQL configuration you can skip the first 4 steps.

1. Download and install Apache web server in your system.

2. Download and install PHP script engine in your system.

3. Download and install mySQL database server in your system.

Configure its access permissions properly. Please refer to the documention of mySQL.

4. It might be necessary to configure the Apache web server to run PHP scripts as CGI.
Please refer to the documentation of Apache in doing this configuration.

5. Download the CARE 2002 beta 1.0.03 scripts in ZIP format (if you have not done it yet).

6. Uncompress the CARE 2002 beta 1.0.03 scripts into the active directory of Apache web server.

If you decide to use a different directory name, please refer to the documentation of Apache in configuring the active directory .

7. Open the file "inc_remoteservers_conf.php" in the directory "/global_conf". Enter the domain or ip addresses and hosts information.

Note: If the web server is within a network enter specifically the IP adresses for the hosts. Never use "localhost".

8. Start the apache web server (or any server that you have configured).

9. Start the mySQL database server.

10. Start your browser.

::::::::::::::::::: SOME NOTES BEFORE YOU PROCEED :::::::::::::::::::::::::

  • In the installation, you will be asked to enter the name of the database, database username, database host name, and database password. Please have these information ready before you start the installation.

  • If you choose to use the default values for the database connection, just leave the default values that automatically appear in the fields.

  • You have to decide how the database should be named. (This will not be created automatically)

  • During the creation of the ICD & OPS data, the screen will freeze for about 10 minutes. Do not panic, the system is working on the background. Just be patient and wait. Do not break or cancel this process, do not close your browser. Otherwise, the data would be incomplete.
    NOTE: To users of previous versions. You can avoid this long phase by opting to use the old database. In the installation options on the screen, follow this options path: -> remove old database -> delete the tables except the ICD/OPS.

  • After you browser comes back to life, you will be asked to enter your Admin access information.

  • You will be asked to enter 3 keys. If you decide to let the installer create random keys automatically, just leave the key fields empty.

    ::::::::::::::::::: OK GO AHEAD NOW :::::::::::::::::::::::::

    11. In your browser, type the following url address: http://?????????/install/install.php .

    (where ???????? is the actual host address of your Apache web server).
    The installation script will start.

    NOTE: Do not type the initialize.php script!!!

    12. Then..., just follow the instructions on the screen.

    13. If everything goes well (hopefully), the installation should be finished after about 10 minutes.

    ::::::::::::::::::: SOME NOTES AFTER :::::::::::::::::::::::::

    VERY IMPORTANT!!!! The install.php script and its auxilliary scripts are not "script locked". After a successful installation it is very important to move the "install" directory and all the scripts contained in it outside the active directory of the webserver. You can of course delete this directory (to be sure). If you do not do this, somebody could run this script and start a new installation, erasing all your data!!

  • If the installation fails (hopefully not), try to reinstall everything.

  • If nothing did work, try installing the package a'la classique (manually). You can read the instructions for manual installation here.
